# Looker Integration Strategy: ML-Enriched Dashboard

This document provides the SQL strategy to join historical price data from the GCP Hub with the signals generated by the Crypto ML Predictor.

---

## 1. Data Schema Overview
*   **Source A (Price Data):** `analytics_data_gold.crypto_historical_trends`
*   **Source B (ML Signals):** `analytics_data_gold.crypto_ml_predictions`

---

## 2. Recommended SQL Join
To create a unified view in Looker Studio or BigQuery, use the following `LEFT JOIN`. This ensures that even if a prediction is missing for a specific day, the historical price remains visible.

```sql
SELECT 
    t.id,
    t.ds,
    t.price,
    t.total_volume,
    t.market_cap_dominance,
    -- ML Signals
    p.zscore_30d,
    p.price_vs_ma30,
    p.ml_prediction,
    p.ml_probability,
    -- Label for Visualization
    CASE 
        WHEN p.ml_prediction = 1 THEN 'BULLISH (BUY)'
        WHEN p.ml_prediction = 0 THEN 'BEARISH/STABLE (WAIT)'
        ELSE 'NO SIGNAL'
    END as ml_signal_label
FROM 
    `your-project-id.analytics_data_gold.crypto_historical_trends` as t
LEFT JOIN 
    `your-project-id.analytics_data_gold.crypto_ml_predictions` as p
ON 
    t.id = p.id AND t.ds = p.ds
ORDER BY 
    t.ds DESC
```

---

## 3. Visualization Best Practices in Looker
*   **Trend Line:** Map `price` to the Y-axis and `ds` to the X-axis.
*   **Heatmap:** Use `zscore_30d` as a color dimension to highlight "Volume Shocks" on the price line.
*   **Probability Filter:** Add a slider for `ml_probability` to allow users to filter for only "High Confidence" signals (> 70%).
